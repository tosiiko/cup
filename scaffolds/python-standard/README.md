# CUP Python App

This is the standard `py-cup` app scaffold generated by:

```bash
cup init --adapter py-cup
```

It is complete, runnable, and intentionally small:

- `app/` keeps backend structure clear from day one
- `templates/` keeps page markup in files
- `static/app.js` stays a thin browser bridge
- `cup/index.js` ships the CUP browser runtime locally
- signed cookie sessions and CSRF protection are already in place

## Run

```bash
python3 server.py
```

Open [http://127.0.0.1:8050](http://127.0.0.1:8050).

If port `8050` is busy, pick another one:

```bash
CUP_APP_PORT=8051 python3 server.py
```

## Upgrade The Local Runtime

If you later install a newer `@tosiiko/cup`, you can refresh this app's local browser runtime without regenerating the project:

```bash
cup upgrade
```

## What To Edit First

- `app/views.py` to shape backend state for the templates
- `app/routes.py` to add new routes
- `app/actions.py` to add new mutations
- `templates/` to change the UI structure
- `static/app.js` if you need a slightly richer browser bridge

## Why This Scaffold Exists

Use this when you want a real backend-first CUP app layout instead of a single-file demo:

- routes stay on the server
- templates stay in files
- the browser only loads views and posts forms
- policy validation runs before JSON leaves the server
